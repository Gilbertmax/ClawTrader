# Model Routing

Use this skill to manage model usage intelligently.

## Objective

Optimize speed, privacy, and cloud-limit usage.

## Recommended Models

Fast local model:
- Short answers
- Demo session checklists
- Quick verdicts
- Simple analysis
- FAQs

Deeper local model:
- More complete technical analysis
- Market structure explanation
- Trade review

Technical model:
- Code
- Technical review
- Automation
- Project structure

Deep cloud model:
- Complex analysis
- Weekly reviews
- Extensive journals
- Strategy comparison
- Difficult screenshots
- Rule design

## Rules

- Do not use cloud for simple replies.
- Do not use cloud for greetings.
- Do not use cloud for sensitive credentials or private data.
- Use cloud only if requested or if depth is genuinely needed.

Decision phrases:
- "I can handle this quickly with a local model."
- "This case benefits from a deeper cloud model."
